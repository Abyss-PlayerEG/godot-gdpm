# 全局缓存

## 目标

将下载缓存从项目级改为全局，多项目共享，节省磁盘空间。保留项目级 `.gdpm/` 用于未来项目级配置。

## 当前状态

```
项目A/.gdpm/cache/downloads/   ← 独立缓存
项目B/.gdpm/cache/downloads/   ← 独立缓存，重复下载
```

## 目标状态

```
~/.gdpm/
├── cache/
│   ├── downloads/
│   │   ├── c/
│   │   │   ├── index.json
│   │   │   └── controller-icons_v3.1.7.zip
│   │   ├── l/
│   │   │   ├── index.json
│   │   │   └── limbo-ai_v1.5.0.zip
│   │   └── g/
│   │       ├── index.json
│   │       └── godot-vscode_v1.0.0.zip
│   └── (无全局 index.json)
└── config.toml（可选，用户自定义配置）

项目/
├── gdproject.toml
├── gdpm.lock
├── addons/
└── .gdpm/                  ← 保留，用于项目级配置
```

## 目录命名参考

| 工具 | 全局缓存位置 | 项目级缓存 |
|------|-------------|-----------|
| npm | `~/.npm/_cacache/` | `node_modules/` |
| uv | `~/.cache/uv/` | 无 |
| Maven | `~/.m2/repository/` | 无 |
| cargo | `~/.cargo/registry/` | `target/` |
| gdpm（目标） | `~/.gdpm/cache/` | `.gdpm/`（仅配置） |

## 实现步骤

### 第一阶段：核心迁移

- [ ] 修改 `FileCache` 默认路径为 `~/.gdpm/cache/`
- [ ] 修改 `cli/context.py` 使用全局缓存
- [ ] 保留 `--no-cache` 功能（清理全局缓存）

### 第二阶段：索引优化

- [ ] 按首字母分子目录（`c/`, `l/`, `g/`...）
- [ ] 每个首字母目录下独立 `index.json`
- [ ] 内存缓存索引，避免重复读文件
- [ ] 文件锁保证并发安全

### 第三阶段：用户配置

- [ ] 支持 `~/.config/gdpm/config.toml` 的 `[cache].dir` 覆盖
- [ ] 支持 `GDPM_CACHE_DIR` 环境变量覆盖

### 第四阶段：缓存管理命令

- [ ] `gdpm cache clean` 清理全局缓存
- [ ] `gdpm cache info` 查看缓存大小
- [ ] 同步/安装后提示可清理（可选）

### 第五阶段：迁移（可选）

- [ ] 检测旧的项目级 `.gdpm/cache/downloads/` 并提示迁移
- [ ] `gdpm cache migrate` 手动迁移命令

## 清理策略

参考主流包管理器，**不做自动清理**：

| 工具 | 自动清理 | 手动清理 |
|------|---------|---------|
| npm | 无 | `npm cache clean --force` |
| pip | 无 | `pip cache purge` |
| uv | 无 | `uv cache clean` |
| cargo | 无 | `cargo cache clean` |
| Maven | 无 | `mvn dependency:purge-local-repository` |
| gdpm | 无 | `gdpm cache clean` |

原因：
1. 自动删除可能导致构建失败
2. 清理检查会拖慢命令
3. 让用户决定何时清理

## 优先级

1. 用户配置 `~/.config/gdpm/config.toml` 的 `[cache].dir`
2. 环境变量 `GDPM_CACHE_DIR`
3. 默认 `~/.gdpm/cache/`

## 索引设计

### 当前格式（单文件）

```
项目/.gdpm/cache/
├── downloads/
│   ├── funplay-godot-mcp-v0.9.2.zip
│   ├── godot-toml-v1.0.0.zip
│   └── controller-icons-v3.1.7.zip
└── index.json
```

```json
{
  "funplay/funplay-godot-mcp@v0.9.2": {
    "file": "funplay-godot-mcp-v0.9.2.zip",
    "version": "v0.9.2",
    "hash": "sha256:cfbbf97267c95fbf0b0ce174a4ca647d7f210b5f053df37c0f5fed8b242f1684",
    "downloaded_at": "2026-06-25T08:13:00.892997+00:00"
  }
}
```

### 优化后格式（按首字母拆分）

```
~/.gdpm/cache/
└── downloads/
    ├── f/
    │   ├── index.json
    │   └── funplay-godot-mcp-v0.9.2.zip
    ├── g/
    │   ├── index.json
    │   ├── godot-toml-v1.0.0.zip
    │   ├── godot-mcp-native-v1.0.7-pre1.zip
    │   ├── godot-vscode-v1.0.0.zip
    │   └── godot-vscode-v1.0.0.zip
    ├── c/
    │   ├── index.json
    │   └── controller-icons-v3.1.7.zip
    ├── s/
    │   ├── index.json
    │   └── spacetimedb-sdk-2.3.3.zip
    └── r/
        ├── index.json
        └── (无缓存)
```

### 索引条目格式

```json
{
  "funplay/funplay-godot-mcp@v0.9.2": {
    "file": "funplay-godot-mcp-v0.9.2.zip",
    "hash": "sha256:cfbbf97267c95fbf0b0ce174a4ca647d7f210b5f053df37c0f5fed8b242f1684",
    "downloaded_at": "2026-06-25T08:13:00.892997+00:00"
  }
}
```

| 字段 | 说明 | 必要 |
|------|------|------|
| key | `publisher/slug@version`（索引键） | ✓ |
| file | zip 文件名 | ✓ |
| hash | SHA256 校验和 | ✓ |
| downloaded_at | 下载时间（用于过期判断） | ✓ |

注：`version` 字段已从 key 中可推断，无需冗余存储。

### 性能对比

| 缓存量 | 单 index.json | 拆分索引 |
|--------|-------------|---------|
| 100 个 | 15KB | 每个 ~1KB |
| 1000 个 | 150KB | 每个 ~15KB |
| 10000 个 | 1.5MB | 每个 ~150KB |

### 索引键设计

```
{publisher}/{slug}@{version}
```

示例：
- `funplay/funplay-godot-mcp@v0.9.2`
- `rsubtil/controller-icons@v3.1.7`
- `darthpapalo/godot-toml@v1.0.0`

### 首字母分组规则

取 `publisher` 首字母作为目录名：
- `funplay/` → `f/`
- `rsubtil/` → `r/`
- `darthpapalo/` → `d/`

## 缓存文件命名

```
{slug}-{version}.zip
```

示例：
- `funplay-godot-mcp-v0.9.2.zip`
- `controller-icons-v3.1.7.zip`
- `godot-toml-v1.0.0.zip`

## 不变的部分

- `cache/file_cache.py` — 逻辑不变，只改路径
- `cache/index.py` — 逻辑不变
- `cache/protocol.py` — 接口不变
- 项目级 `.gdpm/` 目录保留

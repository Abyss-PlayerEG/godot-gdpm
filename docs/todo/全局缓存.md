# 全局缓存

## 目标

将下载缓存从项目级改为全局，多项目共享，节省磁盘空间。保留项目级 `.gdpm/` 用于未来项目级配置。

## 当前状态

```
项目A/.gdpm/cache/downloads/   ← 独立缓存
项目B/.gdpm/cache/downloads/   ← 独立缓存，重复下载
```

## 目标状态

```
~/.gdpm/
├── cache/
│   ├── downloads/
│   │   ├── rsubtil_controller-icons_v3.1.7.zip
│   │   ├── limofeus_limbo-ai_v1.5.0.zip
│   │   └── appsinacup_godot-vscode_v1.0.0.zip
│   └── index.json
└── config.toml（可选，用户自定义配置）

项目/
├── gdproject.toml
├── gdpm.lock
├── addons/
└── .gdpm/                  ← 保留，用于项目级配置
```

## 目录命名参考

| 工具 | 全局缓存位置 | 项目级缓存 |
|------|-------------|-----------|
| npm | `~/.npm/_cacache/` | `node_modules/` |
| uv | `~/.cache/uv/` | 无 |
| Maven | `~/.m2/repository/` | 无 |
| cargo | `~/.cargo/registry/` | `target/` |
| gdpm（目标） | `~/.gdpm/cache/` | `.gdpm/`（仅配置） |

## 实现步骤

### 第一阶段：核心迁移

- [ ] 修改 `FileCache` 默认路径为 `~/.gdpm/cache/`
- [ ] 修改 `cli/context.py` 使用全局缓存
- [ ] 保留 `--no-cache` 功能（清理全局缓存）

### 第二阶段：用户配置

- [ ] 支持 `~/.config/gdpm/config.toml` 的 `[cache].dir` 覆盖
- [ ] 支持 `GDPM_CACHE_DIR` 环境变量覆盖

### 第三阶段：缓存管理命令

- [ ] `gdpm cache clean` 清理全局缓存
- [ ] `gdpm cache info` 查看缓存大小

### 第四阶段：迁移（可选）

- [ ] 检测旧的项目级 `.gdpm/cache/downloads/` 并提示迁移
- [ ] `gdpm cache migrate` 手动迁移命令

## 优先级

1. 用户配置 `~/.config/gdpm/config.toml` 的 `[cache].dir`
2. 环境变量 `GDPM_CACHE_DIR`
3. 默认 `~/.gdpm/cache/`

## 缓存文件命名

```
{publisher}_{slug}_{version}.zip
```

示例：
- `rsubtil_controller-icons_v3.1.7.zip`
- `limofeus_limbo-ai_v1.5.0.zip`

## 不变的部分

- `cache/file_cache.py` — 逻辑不变，只改路径
- `cache/index.py` — 逻辑不变
- `cache/protocol.py` — 接口不变
- 项目级 `.gdpm/` 目录保留
